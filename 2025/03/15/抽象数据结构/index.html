景点抽象数据结构

景点抽象数据结构

景点信息(Info)

景点自身包含以下信息：

  • id : 景点的唯一标识符
  • name: 景点的名称
  • type: 景点的类型（如使用A代表学校，B代表景区）
  • heat: 景点的热度（0-100）
  • rating: 景点的评分（0-5）

建筑物 (Buildings)

每个建筑物包含以下属性：

  • id: 建筑物的唯一标识符
  • name: 建筑物的名称
  • type: 建筑物的类型（如教学楼、宿舍楼、厕所等）

示例：

道路 (Roads)

每条道路包含以下属性：

  • id: 道路的唯一标识符
  • start_building_id: 道路起点建筑物的ID
  • end_building_id: 道路终点建筑物的ID
  • length: 道路的长度
  • access: 道路的通行类型（”A” 表示只有人可以通行的路，”B” 表示人和车都可以通行的路）

图结构 (Graph Structure)

  • 节点 (Nodes): 表示建筑物
  • 边 (Edges): 表示道路，包含起点、终点和长度
